# Dynamic Pricing System

ML-система прогнозирования спроса и динамического ценообразования для российского ритейла на основе исторических данных продаж компании **1С**.

Проект разрабатывается в рамках выпускной квалификационной работы (РТУ МИРЭА).

Система включает:

- Модель прогнозирования спроса (Demand Forecasting)
- Модуль динамического ценообразования (Dynamic Pricing)

## Данные

Используется открытый датасет **Predict Future Sales** (1С), доступный на Kaggle:  
[Competition: Predict Future Sales](https://www.kaggle.com/c/competitive-data-science-predict-future-sales)

Датасет содержит:

- ежедневные продажи товаров (количество, цена);
- справочники товаров, категорий и магазинов;
- временные метки транзакций.

После предобработки данные агрегируются до месячного уровня:  
`shop_id | item_id | month | sales_qty | revenue | avg_price | main_category | super_category`

## Exploratory Data Analysis (EDA)

Проведён исследовательский анализ данных:

- структура и размер датасета;
- распределения продаж и цен;
- основные товарные категории и их вклад в выручку;
- сезонность и тренды;
- выбросы и аномалии;
- зависимость спроса от цены (эластичность);
- жизненный цикл товаров.

Все графики автоматически сохраняются в `artifacts/figures/eda/`.

## Data Cleaning

Реализован полный пайплайн предобработки:

- объединение таблиц продаж, товаров, категорий, магазинов;
- преобразование дат и выделение месяца;
- проверка пропусков (отсутствуют);
- удаление дубликатов и некорректных значений;
- агрегация дневных продаж до месячного уровня;
- создание укрупнённых категорий (`main_category`, `super_category`);
- сохранение результата в **Apache Parquet** (`data/interim/monthly_sales.parquet`).

Parquet обеспечивает компактное хранение, сохранение типов и быстрое чтение.

## Feature Engineering (план)

Будут созданы следующие группы признаков:

- **Временные лаги:** продажи за предыдущие 1, 2, 3, 12 месяцев.
- **Скользящие статистики:** среднее, медиана, тренд за 3/6/12 месяцев.
- **Ценовые признаки:** относительное изменение цены, скидка, цена относительно средней по категории.
- **Категориальные взаимодействия:** shop × category, item × month.
- **Календарные признаки:** месяц, квартал, индикаторы праздников.
- **Макро-признаки:** инфляция, курс валют (если доступны).

## Модели прогнозирования спроса

### Baseline
- Наивный прогноз (предыдущий месяц).
- Среднее по товару и месяцу.
- Линейная регрессия на лагах и цене.

### Основные модели (будут протестированы)
- **LightGBM / CatBoost** — градиентный бустинг над табличными признаками.
- **SARIMA** / **Prophet** — для сравнения с классическими временными рядами.
- **LSTM / N-BEATS** (опционально) — глубокое обучение для временных рядов.

### Метрики
- MAE, RMSE, WAPE, sMAPE.
- Бизнес-ориентированная метрика: точность на топ-20% товаров по выручке.

---

## Модуль динамического ценообразования

1. **Генерация кандидатов:** сетка цен вокруг текущей (±10%).
2. **Прогноз спроса** для каждого кандидата обученной моделью.
3. **Оптимизация:**
   - Линейная функция спроса `Q = a - b * P` (коэффициенты оцениваются моделью).
   - Целевая функция: максимизация прибыли / выручки.
   - Ограничения: min/max цена, соблюдение маржинальности.
4. **Результат:** рекомендованная цена для каждой пары товар–магазин на следующий месяц.
